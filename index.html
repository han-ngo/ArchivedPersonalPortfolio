Open Source: Canvas TA Queue


Canvas is a widely-used LMS Platform across the world. We are adding a new core feature – TA Queue – to this open-source software. A TA Queue system available directly within Canvas would provide the benefits to many courses at any schools that use Canvas. This prevents "re-inventing the wheel" situations, where each school or departments has to create their own small TA queue application. Ultimately, the Canvas-integrated TA queue would allow for more efficient and directed TA hours, thus increase student success as a result.

In addition to the basic TA Queue functions, this project also provides:

  • Remote TA help hours via Video Conferences
  • Statistics / Insights Page for Professors and TA
  • Easy Scheduling System

Data Visualization: Uber-Moves


Most people see everyday traffic on their daily routine but often don't know much detail about the overall traffic. This project provides a visual exploration one can use to find out where and when people utilize Uber mostly. It helps the drivers see and visualize pickups in order to place himself in a better location and time for maximum pickups during his work hours.

The Implementation contains:

  • Three different types of data mapping: Cluster, Circles, and Heatmap
  • Circular brush selections to specify days-of-the-week (eg: Mon-Wed) and hours-of-the-day (9am-14pm)
  • Three charts to clearly visualize the pickup pattern.
  • Cluster map provides several clusters which are groups of pickup points. Selecting a particular cluster provides the details of that cluster in a tabular format, organized/sorted by district, date and time.
  • Three preset buttons, showing three different interesting story points.

Learning Outcome Tracker


This project helps the School of Computing to track how well courses are teaching students. This process involves:

  1. Defining specific Learning Outcomes a student should learn for a given course (done by Curriculum Definer role)
  2. Allowing a Professor role to attach Evaluation Metrics (one or more assignments and/or exams) to each LO assigned to the course. This involves the professor saving a copy of the test/assignment on the LO system.
  3. At the end of the semester, the Professor would also upload one or more samples of Student Work for each Evaluation Metric.
  4. After all professors have completed all of their reporting, a Department Chair role can view a page showing the status of completion of for all courses.

Some additional features:

  • Users and Roles Adjustment
  • Student Feedbacks and Course Ranking System
  • Search Bar for Courses
